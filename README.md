# Windows-Server-AD-GPO-Lab

## Objective
Deploy and configure a Windows Server Active Directory environment in order to simulate enterprise identity and access management operations.

## Environment
> Windows Server virtual machine
> Active Directory Domain Services (AD DS)
> Group Policy Management (GPMC)
> Client machine joined to the domain
> Isolated lab environment

## Disclaimer
"This lab was conducted in a controlled and authorized environment strictly for educational purposes."

## Lab Setup
1. Install and configure Windows Server
2. Promote the server to a Domain Controller
3. Configure Active Directory Domain Services
4. Create organizational units (OUs)
5. Create domain users and groups
6. Join a client machine to the domain

## Tools Used
> Windows Server
> Active Directory Users and Computers (ADUC)
> Group Policy Management Console (GPMC)
> Windows Client Machine
> Virtualization platform (VMware)

## Active Directory Configuration
> Creation of domain users and security groups
> Configuration of Organizational Units (OUs)
> User authentication through Active Directory
> Domain-based centralized administration

## Group Policy Configuration
> Creation and deployment of Group Policy Objects (GPOs)
> Application of policies to users and computers
> Restriction and management of system configurations
> Verification of policy enforcement on client machines

## Domain Integration
> Joining a client machine to the Active Directory domain
> Authentication using domain user credentials
> Validation of centralized login functionality
> Communication between domain controller and client machine

## Security Considerations
> Principle of least privilege applied to users
> Centralized authentication and policy management
> Controlled access within the lab environment
> Administrative separation between local and domain accounts
